The Engineering Question

How Does an Agent Get Smarter?

Not "how do you add context to a prompt" — your engineers already know RAG. The question is: what happens when you need an agent that learns from its own mistakes, improves its own routing, extracts reusable rules from experience, and does all of this before it processes the next prompt?

The Naive Approach
RAG

Retrieve, stuff, hope

Semantic search against a vector store. Retrieve top-K chunks. Stuff them into the prompt. Works for Q&A — fails for agents. No notion of what worked vs. what didn't. No utility scoring. No mistake prevention. Every prompt is equally naive.

The Common Approach
Fine-tune

Retrain the model

Collect data, fine-tune weights, deploy a new model. Slow (hours to days), expensive ($100s-$1000s), and brittle — the model forgets previous training (catastrophic forgetting). Can't learn within a session. Can't learn from a single mistake.

What We Built
Loop

7 mechanisms. Real-time. Compounding.

Every prompt triggers 4 parallel analysis threads in <2 seconds. MemRL scores knowledge utility. ExpeL extracts reusable rules. Reflexion indexes domain-specific lessons. A neural router learns from session outcomes. Mistakes are tracked and injected. It never makes the same error twice.

Layer 1 — The Prompt Intelligence Engine

4 Threads. <2 Seconds.

Every time a user hits Enter, a hook fires before the model sees the prompt. Four parallel threads run simultaneously: KB preflight search, skill detection, the learning engine, and deep cognitive reasoning. Their outputs merge into a single intelligence injection — 300-800 tokens of context that transforms a naive prompt into an informed one.

1

KB Preflight

Hybrid search: pgvector cosine similarity + PostgreSQL full-text, fused via Reciprocal Rank Fusion. Returns top domain mistakes, known patterns, and repeat error flags. Runs qwen3:8b as GPU reranker. ~800ms.

2

Skill Detection

First tries the LearnedRouter — an MLX neural network trained on prior session outcomes. If confidence < 0.40, falls back to keyword matching against 55 registered skills. Returns ranked skill matches with confidence scores.

3

Learning Engine

Three sub-engines fire in parallel: ReflexionEngine injects domain-indexed reflections. ExperienceEngine retrieves high-confidence rules. CoreMemoryEngine checks if any self-edit is warranted. Combined output: 100-300 tokens of learned context.

4

Deep Reasoning

Single call to qwen3:8b with a structured system prompt. Returns: intent type (directive/invitation/correction/curiosity/emotional/compound), reasoning mode, emotion detection (frustration-calibrated), and 2-3 curiosity questions. ~400ms.

Layer 2 — The Cognitive Layer

One Call. Five Analyses.

A single call to qwen3:8b (local, on Apple Silicon, zero API cost) returns five distinct analyses in ~400ms. This isn't prompt classification — it's cognitive preprocessing that changes how the agent thinks about the task before it starts.

Intent Parser

What kind of request is this?

6 types: directive (do this), invitation (explore this), correction (you got it wrong), curiosity (I wonder about), emotional (I'm frustrated), compound (multiple intents). Each type triggers different agent behavior — corrections trigger "STOP and acknowledge", invitations trigger "EXPLORE before acting."

Reasoning Mode Selector

What thinking style fits?

9 modes auto-selected per prompt: inversion (what would make this fail?), pre-mortem (imagine it shipped and broke), systems-thinking (second-order effects), first-principles (strip assumptions), analogical (what's this similar to?), and 4 more. Injected as a one-line action directive.

Emotion Recognition

Is the human frustrated?

Returns a 0-1 weight. Calibrated to Rick's communication style (short sentences = frustration, ALL CAPS = emphasis, "..." = thinking). When weight > 0.5: "STOP and acknowledge before reasoning." Prevents the agent from plowing forward when the human needs to be heard.

CognitiveAnalysis (actual dataclass)
@dataclass
class CognitiveAnalysis:
    intent: str          # directive | invitation | correction | curiosity | emotional | compound
    reasoning_mode: str   # inversion | pre-mortem | systems-thinking | first-principles | ...
    curiosity: list[str]  # 2-3 questions to hold while working
    emotion: str          # "frustration(0.7)" or "neutral(0.1)"
    action: str           # One-line directive: "EXPLORE before acting"

# Runs on: qwen3:8b (local, Apple Silicon M4 Max GPU)
# Latency: ~400ms (always hot in VRAM)
# Cost: $0.00 (local inference)
# File: lib/cognitive.py

Why local, not API: The cognitive layer fires on every single prompt. At 400ms per call, it adds negligible latency. At $0.00 per call, it costs nothing. At 8B parameters with thinking mode, qwen3:8b provides sufficient quality for intent classification and emotion detection. Using Claude or GPT-4 for this would add $0.01-$0.05 per prompt — thousands of dollars per month for something a local model handles perfectly. Reserve API calls for reasoning, not classification.

Curiosity Checkpoints (Mid-Session)

Every 20 tool calls, a static checkpoint fires — no LLM call, zero latency. 7 rotating questions force the agent to pause and check itself:

Checkpoint Rotation (static, no LLM)
"Am I still reasoning, or pattern-matching?"
"What assumption am I making that I haven't verified?"
"If I were wrong about my current approach, what would I see?"
"Am I solving the right problem, or the easiest one?"
"What would Rick's reaction be if he saw my work right now?"
"Is this making the system better, or just different?"
"What context might I be missing from another session?"
Layer 3 — Seven Mechanisms for Getting Smarter

The Compound Engine.

Each mechanism addresses a different aspect of learning. Together, they create a system where every interaction improves future interactions — not linearly, but compoundingly. A mistake prevented today prevents the same mistake in every future session. A rule extracted today accelerates every future decision.

1
MemRL — Google DeepMind

Utility-Scored Memory

Every KB entry has a utility_score updated via Exponential Moving Average. When knowledge helps → score increases. When retrieved but unhelpful → score decays. Temporal decay prevents stale knowledge from dominating. High-utility entries surface first in search.

2
ExpeL — NeurIPS 2023

Rule Extraction from Experience

After task completion, the ExperienceEngine extracts reusable rules: "Separate deployments = asymmetric keys". Rules carry a confidence score (0-1). Successful application → confidence ×1.05. Failed application → confidence ×0.9. Top rules inject into preflight. Ranked by confidence × helpfulness.

3
Reflexion — NeurIPS 2023

Domain-Indexed Self-Reflection

ReflexionEngine stores reflections indexed by domain. When a new prompt arrives in a known domain, the 3 most recent reflections inject automatically. reflection_index table: domain, lesson, context, created_at. Guarantees minimum 3 reflections per domain context.

4
Code Review Learning

Bidirectional KB Integration

Every code review teaches the system. New finding types → ingested to KB tagged "code-review". Recurring findings (3+ occurrences) → auto-escalated to HIGH priority. Resolved findings → fix pattern documented. KB known issues → flagged as "KNOWN REPEAT" in reviews. lib/review_learning.py.

5
Neural Router — MLX

Learned Skill Routing

An MLP trained on Apple Silicon via MLX: 1024→256→64→12. Input: prompt embedding (mxbai-embed-large). Output: skill routing scores. Trained on the routing_decisions table — every session's AES (Agent Effectiveness Score) provides the feedback signal. Replaces keyword matching with learned behavior.

6
Dynamic Checklists

Domain-Specific Enrichment

Preflight detects the domain (auth, database, API, etc.) and enriches the checklist with domain-specific items from KB history. Includes reasoning traps — common mistakes specific to each domain. 17 repeat error themes tracked. If the current prompt matches a theme, the agent gets an explicit warning.

7
CoreMemoryEngine

Safe Self-Editing

The nuclear option: the system can edit its own operating instructions (CLAUDE.md). Tightly constrained — only appends to designated sections, never deletes, requires high-confidence justification. Used for permanent behavioral changes that survive all future sessions. The agent literally rewrites its own rules.

utility_score = EMA(score, feedback, α=0.15) × temporal_decay(age, rate=0.005, floor=0.6)
MemRL Utility Scoring — Exponential Moving Average with Temporal Decay Floor

Why 7 mechanisms, not 1: Each mechanism addresses a different learning timescale. MemRL operates on individual retrievals (milliseconds). ExpeL operates on task completion (minutes). Reflexion operates on domain experience (hours). The Neural Router operates on session outcomes (days). Code review learning operates on codebase evolution (weeks). Dynamic checklists operate on error pattern accumulation (months). CoreMemory operates on permanent behavioral change (never expires). Together, they cover every timescale of learning.

Layer 4 — The Knowledge Architecture

PostgreSQL + pgvector + WAL.

The knowledge base isn't a document store. It's a scored, searchable, self-improving memory system with zero-loss writes, hybrid retrieval, and GPU-accelerated reranking. Every piece of knowledge carries metadata about how useful it's been, how often it's retrieved, and when it was last relevant.

Knowledge Table Schema (PostgreSQL + pgvector)
CREATE TABLE knowledge (
    id            SERIAL PRIMARY KEY,
    title         TEXT NOT NULL,
    content       TEXT NOT NULL,
    category      TEXT,          -- identity | operational | patterns | mistakes | ...
    scope         TEXT,          -- system | family | personal | child
    tags          TEXT[],
    source        TEXT,          -- filename for upsert dedup
    embedding     vector(1024), -- mxbai-embed-large (local, free)
    utility_score FLOAT DEFAULT 1.0,     -- MemRL EMA
    retrieval_count INT DEFAULT 0,        -- popularity signal
    created_at    TIMESTAMPTZ DEFAULT NOW(),
    updated_at    TIMESTAMPTZ DEFAULT NOW(),
    owner_id      UUID REFERENCES users(id)  -- RLS enforcement
);

-- Indexes
CREATE INDEX ON knowledge USING ivfflat (embedding vector_cosine_ops);
CREATE INDEX ON knowledge USING gin (to_tsvector('english', title || ' ' || content));
CREATE INDEX ON knowledge (category, utility_score DESC);

Hybrid Search with Reciprocal Rank Fusion

Semantic Search

pgvector cosine similarity

Query → embedding via mxbai-embed-large (local). 1 - (embedding <=> query_vec) as cosine similarity. Returns top-20 candidates. Captures meaning — "authentication" matches "login system" even without keyword overlap.

Full-Text Search

PostgreSQL tsvector

ts_rank_cd() on title+content. Returns top-20 candidates. Captures exact terms — "RS256" matches "RS256" without needing embedding proximity. Critical for code, error messages, and technical identifiers.

RRF Fusion + GPU Rerank

Reciprocal Rank Fusion → qwen3:8b

RRF(d) = Σ 1/(k + rank_i(d)) with k=60. Merges both ranked lists without score normalization (scale-invariant). Top-10 fused results → qwen3:8b reranker on Apple Silicon GPU. Final top-5 returned. Best of both worlds.

Write-Ahead Log — Zero Knowledge Loss

KB WAL Protocol (lib/kb_wal.py)
// Every knowledge write goes through the WAL first

1. WRITE: Append entry to pending.jsonl
   → Content-hash dedup: SHA256(title + content + category)
   → If hash exists in pending.jsonl → skip (idempotent)
   → Entry: { title, content, category, scope, tags, hash, timestamp }

2. REPLAY: Process pending.jsonl → PostgreSQL
   → For each entry: call add_knowledge()
   → add_knowledge does upsert by source filename
   → Generates embedding via mxbai-embed-large
   → Inserts or updates row in knowledge table
   → On success: mark entry as replayed

3. RECOVER: On startup, replay any un-replayed entries
   → Handles: crash during write, DB down, embedding failure
   → Guarantee: if it hit pending.jsonl, it will reach PostgreSQL
   → Zero knowledge loss. Period.
Engineering Pattern
Write-Ahead Logging — Same pattern as PostgreSQL, SQLite, Kafka
The WAL pattern is proven at every scale: PostgreSQL uses WAL for crash recovery, SQLite uses WAL for concurrent writes, Kafka uses append-only logs for durability. Applied to a knowledge base, it means the agent can crash mid-thought, restart, and never lose a single piece of learned knowledge. The pending.jsonl file is the single source of truth until replay confirms persistence.

The Compounding Effect — Why This Changes Everything

Session 1 vs. Session 1,000.

The learning loop doesn't add value linearly. Each mechanism feeds the others. Mistakes prevented create positive compounding — the system avoids entire classes of errors, freeing capacity for novel problems. Here's what changes over time.

Session 1 — Cold Start
Raw prompts, no context, keyword-only routing
KB is empty or minimal. Preflight returns nothing useful. Skill routing uses keyword heuristics only (no trained router). Cognitive layer fires but has no prior reflections to inject. Every decision is from first principles. Error rate is highest here.
Sessions 10-50 — Pattern Formation
KB accumulates. First rules extracted. Repeat errors tracked.
MemRL starts differentiating useful vs. useless knowledge. ExpeL extracts the first reusable rules. The 17 repeat-error themes begin populating. Code reviews start teaching the system about recurring findings. Preflight returns relevant warnings for the first time.
Sessions 50-200 — Router Training
Neural router has enough data to train. Keyword fallback decreasing.
The routing_decisions table has hundreds of entries with AES scores. MLX training produces a router that outperforms keyword matching. Skill detection shifts from heuristic to learned. ExpeL rules reach critical mass — most common domains have 5-10 high-confidence rules each.
Sessions 200-500 — Institutional Memory
The system knows more than any individual session can hold.
KB has thousands of entries with calibrated utility scores. Temporal decay has naturally pruned stale knowledge. The 3 most recent reflections per domain are consistently relevant. Code review findings auto-escalate recurring issues. The system prevents errors the current agent has never seen — because a previous session learned the lesson.
Sessions 500+ — Compound Intelligence
The learning loop is the competitive moat.
Every new session starts with the accumulated intelligence of all prior sessions. The neural router accurately predicts skill requirements. ExpeL rules cover most domains. MemRL has naturally surfaced the highest-value knowledge. The system's effective intelligence is the sum of every session that preceded it — and it keeps growing. This is the product.
Strategic Insight
Compounding Knowledge as Competitive Moat
Models commoditize. Prompts commoditize. RAG pipelines commoditize. But accumulated institutional knowledge — scored, validated, self-improving — does not commoditize. Every customer interaction makes the learning loop smarter. Every mistake prevented increases the switching cost. A competitor can copy the architecture in weeks. They cannot copy thousands of sessions of accumulated, utility-scored, domain-specific knowledge. The moat compounds with usage.
